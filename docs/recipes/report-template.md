# 리포트형 레시피 템플릿

영문 문서는 [report-template-en.md](report-template-en.md)를 참고하세요.

이 템플릿은 Creator Hub 레시피를 단순 알림이 아니라 운동 분석 리포트처럼 만들 때 사용합니다.

## 기본 구조

```md
# <리포트 이름>

## 한 줄 판정

이번 기간의 핵심 상태를 한 문장으로 씁니다.

예: 이번 주는 부하가 낮은 회복성 주간입니다.

## 핵심 KPI

| 지표 | 값 | 비교 |
|---|---:|---|
| 주간 부하 | 344 | 직전 7일 대비 +18% |
| 거리 | 182km | 직전 7일 대비 -6% |
| 운동 시간 | 7시간 12분 | 활동 4회 |

## 도표

- 일별 load 막대 차트
- 전주 대비 거리/시간/load 변화
- 선택: CTL/ATL/TSB, zone time, 고도 분포

## 대표 활동

- 가장 긴 활동
- 가장 높은 부하 활동
- 선택: 본인 private 리포트의 지도/고도 썸네일

## 다음 행동

1. 다음 24-48시간 권장 강도
2. 다음 고강도 전 확인할 회복 신호
3. 다음 리포트에서 다시 확인할 지표

## 공유용 요약

경로, 출발/도착 위치, 활동명, 상세 심박/파워 원자료를 제거한 문장만 둡니다.
```

## HTML 카드 규칙

- 상단은 제목과 한 줄 판정만 둡니다.
- KPI는 3-4개 카드로 제한합니다.
- 차트는 숫자를 보조하는 용도입니다. 장식용 그래픽만 넣지 않습니다.
- 지도/썸네일은 본인 private 리포트에만 넣습니다.
- 공개용 리포트에는 집계 차트와 날짜 단위 힌트만 사용합니다.

## 데이터 안전 기본값

| 항목 | 기본값 |
|---|---|
| 활동명 | 제외 |
| 출발/도착 위치 | 제외 |
| 경로 좌표 | 제외 |
| 지도/경로 썸네일 | private report에서만 선택 |
| 심박/파워 상세 스트림 | 제외 |
| 집계 거리/시간/상승고도/load | 포함 가능 |
| 날짜 단위 힌트 | 포함 가능 |

## 외부 개발자 PR에 필요한 것

- 필요한 scope
- API 호출 순서
- sample input/output
- 리포트 스크린샷 또는 HTML 예시
- private report와 public summary 구분
- 자동 실행 주기
- 실패/재시도 정책
- API key와 외부 서비스 secret 보관 방법
