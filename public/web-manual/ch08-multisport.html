8. 멀티스포츠 — 러닝·수영·철인

이 섹션의 목적

지금까지의 설명은 모두 자전거 기준이었습니다. Orider 웹은 러닝·수영·철인(트라이애슬론)도 지원합니다. 이 장에서는 각 종목이 자전거와 무엇이 다른지 — 파워 대신 페이스, FTP 대신 임계 페이스/CSS — 를 종목별로 정리합니다.

핵심 개념은 자전거와 동일합니다. 부하(TSS)를 쌓아 CTL(체력)·ATL(피로)·TSB(컨디션)를 추적하고, 목표 이벤트를 향해 계획을 세웁니다. 달라지는 것은 강도를 측정하는 기준 지표 하나입니다.

구분자전거러닝수영
강도 기준파워 (W)페이스 (분/km)페이스 (분/100m)
임계값(역치)FTP (W)임계 페이스 (LTHR 기반)CSS (임계 수영 속도)
부하 표기TSSrTSSsTSS
존(Zone) 기준파워 존페이스 존 (임계 페이스 기준 Z1~Z5)페이스 존 (CSS 기준 Z1~Z5)
고유 분석NP·IF·파워커브km 스플릿·GAP·케이던스세트·SWOLF·스트로크
자전거에서 FTP가 했던 역할을 러닝은 임계 페이스가, 수영은 CSS가 대신합니다. 이 값을 프로필에 설정하면 존·강도·목표 실현 가능성 판정이 모두 이 기준으로 계산됩니다.

8.1 종목 전환 — 같은 화면, 다른 종목

확인할 것

대시보드·피트니스·목표 설정 화면 상단에 종목 전환 탭(사이클 / 러닝 / 수영)이 있습니다. 탭을 누르면 해당 종목 데이터로 화면이 바뀝니다.

탭나타나는 화면통합(철인) 탭 포함 여부
대시보드해당 종목 활동·통계만 표시없음 (사이클·러닝·수영 3개)
피트니스종목별 CTL/ATL/TSB·존·임계값있음 (통합·사이클·러닝·수영 4개)
목표 설정종목별 목표 위저드없음 (사이클·러닝·수영 3개)
주소창의 ?sport= 값으로 현재 종목을 알 수 있습니다. ?sport=run은 러닝, ?sport=swim은 수영, 파라미터가 없으면 사이클(기본값)입니다. 종목을 바꾸면 다음 접속 시에도 선호 종목으로 기억됩니다.

결과

자전거·러닝·수영을 하나의 계정에서 따로 관리하면서, 종목 탭 하나로 전환할 수 있습니다.

8.2 러닝 — 페이스로 읽는 활동

확인할 것

러닝 활동 상세에는 자전거와 다른 전용 카드가 표시됩니다. 핵심은 속도 대신 페이스(분/km)로 모든 것을 읽는다는 점입니다.

카드읽는 법
km 스플릿 테이블1km 단위로 페이스·심박·케이던스·존을 나열. 가장 빠른 구간에 ★ 표시. 마지막 미완성 구간(1km 미만)은 흐리게 표기
페이스 프로필구간별 페이스 곡선 — 앞이 빠르고 뒤가 느려지면 오버페이스
GAP (경사 보정 페이스)오르내림의 영향을 제거해 "평지였다면 이 페이스" 로 환산. 언덕 코스에서 실제 페이스보다 빠르게 나오면 그만큼 힘을 더 쓴 것
케이던스분당 보수(spm)·보폭 추정. 180 spm 기준선과 비교
rTSS (러닝 부하)이 러닝이 준 총 부하. 50 이하 가벼움 · 100 전후 보통 · 150 이상 장거리/레이스급
GAP(Grade Adjusted Pace)는 경사별 에너지 소모 모델(Minetti)로 계산됩니다. 같은 4:30/km라도 오르막이면 평지 환산 페이스는 더 빨라지므로, 코스가 다른 두 러닝의 강도를 공정하게 비교할 수 있습니다.

임계 페이스와 존: 피트니스 화면 러닝 탭의 핵심 임계값은 임계 페이스(LTHR 기반 추정)입니다. 자전거 FTP에 해당하는 값으로, 이 페이스를 기준으로 5단계 페이스 존이 자동 산출됩니다.

임계 페이스를 프로필에 설정하지 않으면 기본값(4:35/km)으로 존과 목표가 계산됩니다. 자전거의 FTP와 마찬가지로, 임계 페이스를 갱신하면 존과 목표 판정이 즉시 다시 계산됩니다.

결과

"평균 페이스"라는 한 숫자 대신, 구간별 페이스·경사 보정·존 분포로 러닝의 질을 입체적으로 파악합니다.

8.3 수영 — CSS와 100m 페이스

확인할 것

수영은 거리가 짧고 세트(레인) 단위로 진행되므로 분석 단위가 다릅니다. 모든 페이스는 100m당 시간(분/100m)으로 표기됩니다.

카드읽는 법
세트별 스플릿 테이블세트(워밍업/메인/쿨다운)별 거리·시간·100m 페이스·심박·스트로크. 메인 세트는 강조 표시
레인 타임라인세트별 거리·강도를 수영장 레인 형태로 시각화
페이스 디케이메인 세트들의 100m 페이스 변화. 후반에 느려지면 "페이딩", 유지되면 "안정"
스트로크 분석SWOLF · 스트로크 레이트(spm) · DPS(스트로크당 거리)
sTSS (수영 부하)이 수영이 준 총 부하. 30 이하 가벼움 · CSS 강도 · 100 이상 고강도
CSS 갱신 안내이번 활동의 평균 100m 페이스를 보여주며 CSS 갱신 여부를 안내
CSS(Critical Swim Speed, 임계 수영 속도)는 자전거 FTP·러닝 임계 페이스에 대응하는 수영의 역치 지표입니다. 단위는 분/100m이며, 이 값을 기준으로 5단계 페이스 존이 산출됩니다. (기본값 1:48/100m)
수영 강도 분포의 이상적 비율은 저강도(Z1~Z2) 60% 내외 + CSS 이상 고강도 약 20%입니다. 피트니스 수영 탭의 존 분포에서 이 균형을 확인할 수 있습니다.

결과

"몇 미터 수영했는가"를 넘어, 세트별 페이스·SWOLF·CSS 기준 존으로 수영 훈련의 질을 분석합니다.

8.4 철인(트라이애슬론) — 세 종목을 한 화면에

확인할 것

피트니스 화면의 통합 탭은 자전거·러닝·수영을 따로 보지 않고, 세 종목의 부하를 합산해 전체 컨디션을 한눈에 보여줍니다.

요소읽는 법
통합 CTL/ATL/TSB세 종목 부하를 합산한 전체 체력·피로·컨디션. 종목별 CTL이 누적 영역으로 쌓여 표시
종목별 기여도전체 CTL에서 자전거·러닝·수영이 차지하는 비율(도넛)
종목별 드릴다운 카드각 종목의 CTL·주간 TSS·추세. 카드를 누르면 해당 종목 피트니스 화면으로 이동
일별 부하 차트최근 42일의 일별 부하를 종목별로 쌓아 표시 (운동 없는 날은 휴식으로 표기)
주간 TSS 요약이번 주 총 부하와 종목별 분배
통합 뷰는 종목별로 따로 산출한 부하를 같은 날짜축에서 합산합니다. 따라서 "수영은 충분한데 러닝이 부족하다" 같은 종목 간 균형을 한 화면에서 판단할 수 있습니다.
통합 뷰의 부하는 각 종목의 활동에서 계산됩니다. 자전거에만 파워미터가 있고 러닝·수영은 페이스 기반으로 추정되므로, 종목별 부하의 절대값을 1:1로 비교하기보다 각 종목 내부의 추세를 함께 보는 것이 정확합니다.

결과

철인 훈련처럼 세 종목을 병행할 때, 전체 부하와 종목별 균형을 동시에 관리할 수 있습니다.

8.5 종목별 목표 설정

확인할 것

목표 설정 화면에서 종목 탭을 러닝/수영으로 바꾸면 해당 종목 전용 목표 위저드가 나타납니다. 자전거가 코스의 거리·고도로 필요 파워(W/kg)를 판정하는 것과 달리, 러닝·수영은 목표 시간이 임계값 대비 얼마나 빠른가로 실현 가능성을 판정합니다.

구분자전거러닝수영
이벤트 예시코스 선택5K·10K·하프·풀·트레일1500m 테스트·오픈워터·철인 750/1500m
판정 기준필요 W/kg vs 지속 가능 W/kg목표 페이스 vs 임계 페이스목표 100m 페이스 vs CSS
목표 유형완주 / 기록완주 / 기록 / 레이스완주 / 기록 / 기술

실현 가능성은 네 단계로 표시됩니다 — 여유(easy) · 적정(on_track) · 도전(stretch) · 무리(risky). 위저드는 목표일까지의 기간을 빌드업·피크·테이퍼로 나눈 계획과 페이스 존을 함께 미리 보여줍니다.

수영 목표에서 오픈워터(OW)나 철인(Tri) 이벤트를 고르면 수영장 기록 대비 보정 안내가 함께 표시됩니다. 임계 페이스·CSS를 프로필에 설정해 두면 판정이 더 정확해집니다.

결과

자전거와 동일한 방식으로 러닝·수영 목표를 세우고, 목표 페이스가 현재 실력에 맞는지 즉시 확인한 뒤 훈련 계획을 생성합니다.
