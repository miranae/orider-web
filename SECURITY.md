# 보안 정책 / Security Policy

## 취약점 제보
보안 취약점을 발견하면 **공개 이슈(public issue)로 올리지 말고** 아래로 **비공개 제보**해 주세요.

- 이메일: **security@orider.co.kr**

제보에 포함해 주세요:
- 영향 범위와 심각도
- 재현 절차(가능하면 PoC)
- 영향을 받는 버전/커밋

접수 후 합리적 기간 내 회신하며, 수정 전까지 비공개로 협력합니다.

## 범위
이 저장소는 **Orider 웹 프론트엔드(클라이언트)**입니다. 클라이언트는 Firebase 백엔드를 호출하며, **서버측 권한·검증은 백엔드(비공개)와 Firestore 보안 규칙에서 강제**됩니다. 따라서:

- `VITE_FIREBASE_*` 웹 설정 값은 브라우저에 노출되는 **공개 설정**이며 시크릿이 아닙니다(접근 제어는 보안 규칙으로).
- 클라이언트 코드에서 권한 우회가 가능해 보이는 지점이 있다면, 그 자체보다 **서버측 규칙/검증의 미흡** 가능성을 함께 제보해 주세요.

## 하지 말아야 할 것
- 실제 사용자 데이터에 대한 무단 접근·변조
- 서비스 거부(DoS) 테스트
- 타인 계정 대상 테스트
